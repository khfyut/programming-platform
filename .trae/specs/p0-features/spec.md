# 个性化在线编程与答疑系统 - P0级功能产品需求文档

## Overview
- **Summary**: 基于现有系统架构，实现P0级核心功能，包括个性化自适应学习路径、知识点掌握度分析、错题本与强化训练、AI智能辅导能力升级、管理员权限管理、内容资产管理、全局数据分析和系统运维监控。
- **Purpose**: 解决标准化编程学习"一刀切"的痛点，落地开题报告提出的核心研究目标，补全学习全链路闭环和管理员运营管理体系。
- **Target Users**: 学生用户和管理员用户。

## Goals
- 实现个性化自适应学习路径，为用户生成动态调整的专属学习路径
- 落地知识点掌握度与学情深度分析，为个性化推荐提供数据支撑
- 实现错题本与强化训练功能，解决用户"一错再错"的痛点
- 升级AI智能辅导能力，解决答疑精准性不足的问题
- 完善管理员端核心功能，实现系统可运营、可管控、可追溯

## Non-Goals (Out of Scope)
- P1和P2级功能暂不实现
- 社区与互助学习模块暂不开发
- 编程项目实战模块暂不开发
- 批量内容处理工具和系统配置管理暂不实现

## Background & Context
- 现有系统已具备基础的题库、判题、AI答疑和简单的学习记录功能
- 系统使用Java Spring Boot + Vue3 + MySQL技术栈
- 现有接口文档已定义了基础的用户、代码运行、题库、提交判题、AI答疑和学习记录接口

## Functional Requirements

### 用户端P0级功能
- **FR-1**: 个性化自适应学习路径模块
  - 学前能力测评：提供分语言、分方向的入门测评卷
  - 体系化路径拆解：支持Java后端、Python数据分析等主流学习方向
  - 自适应动态调整：基于用户答题情况动态调整后续内容
  - 学习进度可视化：提供路径地图、章节完成度、关卡解锁进度

- **FR-2**: 知识点掌握度与学情深度分析模块
  - 层级化知识图谱构建：基于题库的知识点字段构建四级结构化知识图谱
  - 知识点掌握度量化：将每个知识点分为四个等级，基于多维度量化评分
  - 多维度学情报告：提供周/月学习报告、能力雷达图、高频错误模式分析
  - 薄弱项智能提醒：自动识别掌握度低于阈值的知识点，推送补弱训练计划

- **FR-3**: 错题本与强化训练模块
  - 错题自动归集：用户提交非AC的题目自动归档
  - 错题深度复盘：提供官方题解、知识点讲解、AI专属错误解析
  - 错题重练与复习计划：支持按多维度筛选错题重新作答，基于遗忘曲线生成复习计划
  - 举一反三强化训练：基于错题推送同类型、同难度的题目

- **FR-4**: AI智能辅导能力升级
  - 上下文多轮对话：支持多轮连续对话，保留历史对话上下文
  - 场景化精准绑定：与当前题目、提交记录、错题深度绑定
  - 多维度辅导能力：新增代码调试与错误定位、解题思路分步引导、代码复杂度优化建议、知识点通俗讲解
  - 问答历史优化：新增按题目、知识点分类的问答历史

### 管理员端P0级功能
- **FR-5**: 精细化用户权限与账号管理模块
  - 细分角色与权限体系：新增教师、助教、内容运营、运维管理员等细分角色
  - 账号全生命周期管理：支持账号启用/禁用、密码重置、角色批量分配、班级/学员管理
  - 安全审计：记录所有管理员的敏感操作，支持操作日志查询与审计追溯

- **FR-6**: 内容资产管理升级模块
  - 知识图谱可视化管理：支持知识点的新增/编辑/层级关系配置
  - 课程与学习路径管理：支持学习路径、课程、章节的创建/编辑/发布
  - 学习资源库管理：支持学习资料的上传、分类、关联知识点、可见范围配置
  - 用户内容审核：支持用户发布的题解、圈子帖子的审核、置顶、删除操作

- **FR-7**: 全局数据分析与运营报表模块
  - 平台核心数据大盘：实时展示用户总数、活跃用户、做题总量、提交正确率、AI答疑量等核心指标
  - 多维度数据分析：用户留存与流失分析、全平台知识点掌握度分布、题目/课程内容效果分析、班级教学效果评估报告
  - 数据导出能力：支持各类运营报表、学情报告的Excel导出

- **FR-8**: 系统运维与监控模块
  - 系统运行状态监控：实时展示服务器资源使用率、接口响应指标、代码沙箱运行状态、AI服务可用性
  - 智能告警管理：设置指标阈值，接口超时、服务器过载、服务不可用时自动触发告警
  - 沙箱与日志管理：监控代码运行沙箱资源占用，支持系统日志、错误日志、访问日志的查询与导出

## Non-Functional Requirements
- **NFR-1**: 性能要求
  - 接口响应时间：核心接口响应时间不超过500ms
  - 系统吞吐量：支持1000并发用户
  - 数据处理能力：批量数据导入导出速度快

- **NFR-2**: 安全性要求
  - 数据加密：用户敏感数据加密存储
  - 权限控制：严格的角色权限控制
  - 安全审计：所有敏感操作都有审计日志

- **NFR-3**: 可靠性要求
  - 系统可用性：99.9%
  - 数据一致性：事务处理确保数据一致性
  - 错误处理：完善的错误处理机制

- **NFR-4**: 可维护性要求
  - 代码可读性：符合Java和Vue代码规范
  - 文档完整性：接口文档、设计文档完整
  - 模块独立性：模块化设计，低耦合高内聚

## Constraints
- **Technical**: 
  - 基于现有Spring Boot + Vue3 + MySQL技术栈
  - 保持现有接口的向后兼容性
  - 遵循现有代码风格和架构设计

- **Business**: 
  - 必须在2026.03.31前完成P0级功能开发
  - 严格按照产品规划文档的功能要求实现
  - 确保系统稳定运行，支撑毕业设计答辩

- **Dependencies**: 
  - AI服务API（现有）
  - 代码沙箱服务（现有）
  - Redis缓存（现有）

## Assumptions
- 现有系统架构能够支持新增功能的扩展
- 数据库结构可以通过迁移脚本进行扩展
- 前端框架Vue3能够支持新功能的实现
- AI服务API能够支持多轮对话和场景化绑定

## Acceptance Criteria

### AC-1: 个性化自适应学习路径模块
- **Given**: 用户完成学前能力测评
- **When**: 系统根据测评结果生成初始能力画像
- **Then**: 系统为用户推荐匹配的学习路径，包含相应的知识点单元、章节和关卡
- **Verification**: `programmatic`

### AC-2: 知识点掌握度分析
- **Given**: 用户完成题目提交
- **When**: 系统分析用户答题情况
- **Then**: 系统更新用户的知识点掌握度，并生成学情报告
- **Verification**: `programmatic`

### AC-3: 错题本功能
- **Given**: 用户提交的代码判题结果为非AC
- **When**: 系统自动将该题目加入用户的错题本
- **Then**: 用户可以在错题本中查看、重练错题，并获取针对性的解析
- **Verification**: `programmatic`

### AC-4: AI智能辅导升级
- **Given**: 用户针对特定题目或提交记录提问
- **When**: 系统将问题与上下文信息一起发送给AI
- **Then**: AI返回针对性的解答，支持多轮对话
- **Verification**: `human-judgment`

### AC-5: 管理员权限管理
- **Given**: 管理员登录系统
- **When**: 管理员进行用户权限管理操作
- **Then**: 系统严格按照角色权限控制操作权限，并记录操作日志
- **Verification**: `programmatic`

### AC-6: 内容资产管理
- **Given**: 管理员需要管理学习内容
- **When**: 管理员创建、编辑、发布学习路径和课程
- **Then**: 系统正确保存和展示相关内容
- **Verification**: `programmatic`

### AC-7: 全局数据分析
- **Given**: 管理员访问数据分析模块
- **When**: 系统生成各类数据报表
- **Then**: 报表数据准确，支持多维度分析和导出
- **Verification**: `programmatic`

### AC-8: 系统运维监控
- **Given**: 系统运行过程中
- **When**: 系统资源使用超过阈值
- **Then**: 系统自动触发告警，并提供详细的监控数据
- **Verification**: `programmatic`
