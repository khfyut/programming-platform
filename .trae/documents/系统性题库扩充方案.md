## 三、题目质量规范

### 3.1 题目表述标准
- **清晰性**: 题目描述无歧义，语言简洁明了
- **完整性**: 包含所有必要信息，无需额外假设
- **准确性**: 技术术语使用正确，符合编程规范
- **可读性**: 格式良好，段落清晰，重点突出

### 3.2 答案正确性标准
- **示例正确性**: 样例输入输出必须经过验证
- **边界条件**: 包含边界情况的测试用例
- **完整性**: 覆盖所有可能的输入情况
- **一致性**: 多语言题目答案逻辑一致

### 3.3 难度匹配标准
| 难度 | 时间限制 | 内存限制 | 代码行数参考 | 知识点数量 |
|------|----------|----------|--------------|------------|
| 基础 | 1000-2000ms | 256MB | 10-30行 | 1-2个 |
| 进阶 | 2000-3000ms | 512MB | 30-80行 | 2-4个 |
| 高级 | 3000-5000ms | 1024MB | 80-200行 | 4-6个 |

### 3.4 学习阶段匹配
- **基础题**: 对应学习路径第1-2章
- **进阶题**: 对应学习路径第3-6章
- **高级题**: 对应学习路径第7-10章

## 四、题目审核机制

### 4.1 审核流程
```
题目创作 → 自审 → 提交审核 → 技术审核 → 内容审核 → 难度评估 → 发布
```

### 4.2 审核维度

#### 4.2.1 内容准确性审核
- [ ] 题目描述无歧义
- [ ] 技术术语使用正确
- [ ] 样例输入输出正确
- [ ] 知识点标注准确

#### 4.2.2 难度适宜性审核
- [ ] 难度级别设置合理
- [ ] 时间/内存限制适当
- [ ] 与学习阶段匹配
- [ ] 代码量适中

#### 4.2.3 学习路径匹配度审核
- [ ] 知识点与路径关卡对应
- [ ] 标签与路径主题一致
- [ ] 难度与路径进度协调
- [ ] 可以通过标签智能匹配

### 4.3 审核角色
| 角色 | 职责 |
|------|------|
| **内容审核员** | 检查题目表述、知识点标注 |
| **技术审核员** | 验证答案正确性、测试用例 |
| **教学专家** | 评估难度、学习匹配度 |

### 4.4 审核标准检查表
```markdown
■ 内容审核
□ 题目描述清晰，无歧义
□ 技术术语准确
□ 知识点标注完整
□ 标签设置合理

■ 技术审核
□ 样例输入输出正确
□ 测试用例覆盖全面
□ 边界条件考虑充分
□ 参考答案正确

■ 难度审核
□ 难度级别设置合理
□ 时间限制合适
□ 内存限制合理
□ 与学习阶段匹配
```
