# 题目与学习路径系统性绑定 - 实施计划

## 当前系统分析

### 数据库现状
- **现有题目数量**：16道题目
- **学习路径数量**：20条（10条Java，10条Python）
- **章节总数**：100个章节（20路径 × 5章节）
- **关卡总数**：260个关卡（20路径 × 13关卡）
- **当前关联方式**：使用 `path_level.problem_ids` 字段以逗号分隔的方式关联

### 现有问题
1. 题目数量严重不足（只有16道，需要约300-400道）
2. 使用逗号分隔字符串的关联方式不规范
3. 缺少专门的多对多关联表
4. 缺少题目与学习路径绑定的管理界面

---

## 实施计划

### [ ] 任务1：数据库层面 - 设计并实现多对多关联关系表
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 创建 `path_level_problem` 关联表，实现关卡与题目的多对多关系
  - 表字段包括：id, level_id, problem_id, order_num, create_time
  - 创建适当的索引以优化查询性能
  - 保持向后兼容，暂时保留 `problem_ids` 字段
- **成功标准**：
  - 关联表创建成功
  - 索引创建完成
  - 可以通过SQL查询关卡的题目列表
- **测试要求**：
  - `programmatic` TR-1.1: 执行DESCRIBE path_level_problem查看表结构
  - `programmatic` TR-1.2: 测试INSERT和SELECT操作正常
  - `human-judgment` TR-1.3: 表设计符合数据库范式
- **备注**：先创建表，不立即迁移旧数据

### [ ] 任务2：题库补充 - 生成高质量示例题目
- **优先级**：P0
- **依赖**：任务1
- **描述**：
  - Java题目：按知识点分类，生成约150-200道题目
  - Python题目：按知识点分类，生成约150-200道题目
  - 题目难度分布：简单(40%)、中等(40%)、困难(20%)
  - 知识点覆盖：基础语法、面向对象、集合框架、IO操作、多线程、算法等
  - 每道题目包含完整的测试用例
- **成功标准**：
  - 题库总题数达到300道以上
  - Java和Python题目各150道以上
  - 题目覆盖所有学习路径的知识点
- **测试要求**：
  - `programmatic` TR-2.1: SELECT COUNT(*) FROM problem >= 300
  - `programmatic` TR-2.2: 验证每道题都有测试用例
  - `human-judgment` TR-2.3: 题目质量符合教学要求

### [ ] 任务3：后端开发 - API接口开发
- **优先级**：P1
- **依赖**：任务1, 任务2
- **描述**：
  - 创建实体类 PathLevelProblem
  - 创建Mapper接口和XML映射
  - 开发以下API接口：
    - GET /api/admin/path-level/{levelId}/problems - 获取关卡的题目列表
    - POST /api/admin/path-level/{levelId}/problems - 绑定题目到关卡
    - DELETE /api/admin/path-level/{levelId}/problems/{problemId} - 解绑题目
    - PUT /api/admin/path-level/{levelId}/problems/order - 调整题目顺序
    - GET /api/admin/problems/unbound - 获取未绑定的题目
  - 实现数据验证逻辑
- **成功标准**：
  - 所有API接口正常工作
  - 数据验证通过
  - 可以通过API管理绑定关系
- **测试要求**：
  - `programmatic` TR-3.1: 使用Postman测试所有API端点返回200状态码
  - `programmatic` TR-3.2: 验证数据验证逻辑生效
  - `human-judgment` TR-3.3: API设计符合RESTful规范

### [ ] 任务4：后端开发 - 数据迁移和兼容

## 实施顺序建议

1. **第一阶段**：任务1（创建关联表）+ 任务2（补充题目）- 并行进行
2. **第二阶段**：任务3（API开发）
3. **第三阶段**：任务4（数据迁移）
4. **第四阶段**：任务5（管理界面）+ 任务6（用户界面）- 并行进行

## 风险评估

- **高风险**：题目数量不足 → 需要优先补充题目
- **中风险**：数据迁移可能出错 → 需要备份数据库
- **低风险**：界面开发 → 可以渐进式完善

## 成功指标

- 题库题目数 ≥ 300道
- 所有学习路径的关卡都有题目绑定
- API接口完整可用
- 管理功能完整
- 用户体验良好
