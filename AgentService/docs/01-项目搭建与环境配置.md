# 学习笔记 01：项目搭建与环境配置

## 1. 核心概念
**本质**：Python项目依赖管理与环境隔离

- **虚拟环境(venv)**：给每个项目独立的Python运行环境，避免包版本冲突
- **pip**：Python包管理器，用于安装第三方库
- **镜像源**：PyPI的国内镜像，加速下载

**知识类型**：🔵 通用后端知识（所有Python项目都需要）

## 2. 设计动机
**为什么要虚拟环境？**
- 不同项目依赖不同版本的包
- 避免全局环境污染
- 便于项目迁移和部署

**为什么要固定版本？**
- 不固定版本：`pip install fastapi` 会装最新版
- 三个月后部署，版本变了，代码可能跑不起来
- 固定版本 = 可重现的构建

**反例**：直接把包装到系统Python，不写requirements.txt
- 三个月后换电脑，不知道项目依赖什么
- 团队协作时，别人无法复现你的环境

## 3. 关键结构
```
项目/
├── venv/                  # 虚拟环境（不提交git）
├── requirements.txt       # 依赖清单
└── app/
```

**关键文件**：
- `requirements.txt`：项目"药方"，记录需要什么药、多少剂量
- `venv/`：项目的"私人药房"，只给这个项目用

## 4. 调用链路
```
开发阶段：
  创建venv → 激活venv → pip install → 写代码 → 运行

部署阶段：
  服务器上 → 创建venv → 激活venv → pip install -r requirements.txt → 运行
```

## 5. 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| `pip不是内部命令` | Python环境变量没配好 | 重装Python时勾选"Add to PATH" |
| `No module named 'fastapi'` | 没激活虚拟环境就运行 | 先执行 `venv\Scripts\activate` |
| 下载速度慢 | 使用官方PyPI | 配置国内镜像源 |
| 代理错误`check_hostname` | 系统代理配置问题 | 关闭代理或使用镜像源 |

**最大误区**：把 `venv/` 文件夹提交到git
- 这个文件夹很大（几百MB）
- 而且别人电脑上重建更方便
- 应该提交 `requirements.txt`，不提交 `venv/`

## 6. 可复用模板

**未来每个Python项目都这样做**：
```bash
# 1. 创建项目目录
mkdir 项目名
cd 项目名

# 2. 创建虚拟环境
python -m venv venv

# 3. 激活（Windows）
venv\Scripts\activate

# 4. 安装依赖
pip install xxx
pip install yyy

# 5. 保存依赖清单
pip freeze > requirements.txt
```

**.gitignore 模板**（以后用git时）：
```
venv/
__pycache__/
*.pyc
```

**镜像源配置**（清华）：
```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

## 8. 判断标准

**"什么时候该用虚拟环境？"**
> 只要是一个独立的Python项目，就应该用。没有例外。

**"什么时候该用镜像源？"**
> 在国内开发，默认就应该配置。不要等下载慢才想起。

**【面试高频点】**
**Q：为什么需要虚拟环境？**
**A**：避免不同项目间的依赖冲突。比如项目A需要Django 3.0，项目B需要Django 4.0，虚拟环境让每个项目有独立的包空间，互不干扰。同时通过requirements.txt可以精确复现环境，便于团队协作和部署。
