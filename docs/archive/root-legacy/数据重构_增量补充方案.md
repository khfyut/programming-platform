## 第二阶段：补充参考答案

### 2.1 缺失参考答案的题目清单

```sql
-- 查询需要补充参考答案的题目
SELECT 
    p.id, 
    p.title, 
    p.difficulty,
    p.content,
    p.input,
    p.output,
    p.tags,
    p.knowledge_points
FROM problem p
LEFT JOIN reference_solution rs ON p.id = rs.problem_id
WHERE rs.problem_id IS NULL
ORDER BY p.difficulty, p.id;
```

## 第三阶段：建立知识点关联

### 3.1 知识点映射表

现有知识点体系：
- **数据结构**：数组、链表、栈、队列、树、图、字符串
- **算法技巧**：双指针、滑动窗口、二分查找、DFS、BFS、回溯、分治、贪心、位运算

## 第六阶段：数据验证

### 6.1 完整性检查

```sql
-- 验证参考答案完整性
SELECT 
    '参考答案' as check_item,
    COUNT(*) as total,
    SUM(CASE WHEN rs.problem_id IS NOT NULL THEN 1 ELSE 0 END) as completed,
    CONCAT(ROUND(SUM(CASE WHEN rs.problem_id IS NOT NULL THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2), '%') as percentage
FROM problem p
LEFT JOIN reference_solution rs ON p.id = rs.problem_id;

-- 验证知识点关联完整性
SELECT 
    '知识点关联' as check_item,
    COUNT(*) as total,
    COUNT(DISTINCT pkp.problem_id) as completed,
    CONCAT(ROUND(COUNT(DISTINCT pkp.problem_id) * 100.0 / COUNT(*), 2), '%') as percentage
FROM problem p
LEFT JOIN problem_knowledge_point pkp ON p.id = pkp.problem_id;

-- 验证样例解释完整性
SELECT 
    '样例解释' as check_item,
    COUNT(*) as total,
    SUM(CASE WHEN sample_explanation IS NOT NULL AND sample_explanation != '' THEN 1 ELSE 0 END) as completed,
    CONCAT(ROUND(SUM(CASE WHEN sample_explanation IS NOT NULL AND sample_explanation != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2), '%') as percentage
FROM problem;
```

### 6.2 质量检查

```sql
-- 检查参考答案代码长度（过短可能不完整）
SELECT 
    p.id,
    p.title,
    LENGTH(rs.solution_code) as code_length
FROM problem p
JOIN reference_solution rs ON p.id = rs.problem_id
WHERE LENGTH(rs.solution_code) < 100;

-- 检查知识点关联数量（每题至少1个）
SELECT 
    p.id,
    p.title,
    COUNT(pkp.knowledge_point_id) as kp_count
FROM problem p
LEFT JOIN problem_knowledge_point pkp ON p.id = pkp.problem_id
GROUP BY p.id, p.title
HAVING COUNT(pkp.knowledge_point_id) = 0;
```
