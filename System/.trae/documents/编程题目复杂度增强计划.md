# 编程题目复杂度增强计划（精简版）

## 1. 问题分析

### 当前题目定义过于简单
当前 `Problem` 实体仅包含以下字段：
- `id` - 主键ID
- `title` - 题目标题
- `content` - 题目描述
- `difficulty` - 难度（简单的整数）
- `language` - 编程语言
- `input` - 示例输入
- `output` - 示例输出
- `createTime` / `updateTime` - 时间戳
- `testCases` - 测试用例列表

**存在的问题**：
1. 难度定义过于粗糙，只有简单的整数，无法体现题目复杂度的多维性
2. 缺少执行限制信息（时间、内存）- **判题必需**
3. 缺少题目分类和标签系统 - **题目管理必需**
4. 缺少辅助学习功能（提示、示例解释）- **用户体验必需**
5. 无法满足在线编程平台的基本需求

## 2. 建议的新增字段（精简版）

基于实际需求和用户反馈，建议新增以下核心字段：

### 2.1 执行限制类（判题必需）
| 字段名 | 类型 | 说明 | 示例值 | 优先级 |
|---------|------|------|---------|--------|
| `time_limit` | INT | 时间限制（毫秒） | 1000（1秒） | 高 |
| `memory_limit` | INT | 内存限制（MB） | 256（256MB） | 高 |

### 2.2 分类标签类（题目管理必需）
| 字段名 | 类型 | 说明 | 示例值 | 优先级 |
|---------|------|------|---------|--------|
| `tags` | VARCHAR(500) | 题目标签，逗号分隔 | "动态规划,贪心算法,数组" | 高 |
| `knowledge_points` | VARCHAR(500) | 知识点，逗号分隔 | "动态规划,状态转移,边界处理" | 高 |

### 2.3 辅助学习类（用户体验必需）
| 字段名 | 类型 | 说明 | 示例值 | 优先级 |
|---------|------|------|---------|--------|
| `hints` | TEXT | 提示信息，JSON格式 | [{"step":1,"content":"考虑使用动态规划"}] | 高 |
| `sample_explanation` | TEXT | 示例解释 | "对于输入1和2，输出应该是它们的和3" | 高 |
