# 题目领域分类体系设计方案

## 一、问题分析

### 1.1 当前问题
- **分类不合理**：以编程语言（Java/Python）作为分类标准，忽略了题目的知识本质
- **重复题目**：同一题目在Java和Python中重复出现，造成数据冗余
- **不利于学习**：用户无法按知识领域系统学习，只能按语言碎片化学习
- **维护困难**：修改题目需要同时修改多个语言版本

### 1.2 改进目标
- **领域导向**：以知识领域（算法、数据结构、数据库等）作为分类标准
- **语言无关**：题目本身不依赖特定语言，支持多语言提交
- **便于学习**：用户可以按领域系统学习，也可以对比不同语言实现
- **易于维护**：每个题目只有一个版本，通过`supported_languages`字段支持多语言

## 二、分类体系设计

## 三、数据库设计

### 3.1 创建题目分类表
```sql
CREATE TABLE problem_category (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '分类ID',
    name VARCHAR(50) NOT NULL COMMENT '分类名称',
    code VARCHAR(50) NOT NULL UNIQUE COMMENT '分类代码',
    parent_id BIGINT DEFAULT NULL COMMENT '父分类ID',
    level TINYINT DEFAULT 1 COMMENT '分类层级：1-一级分类，2-二级分类',
    description VARCHAR(500) COMMENT '分类描述',
    icon VARCHAR(100) COMMENT '分类图标',
    sort_order INT DEFAULT 0 COMMENT '排序',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_parent_id (parent_id),
    INDEX idx_code (code)
) COMMENT='题目分类表';
```

### 3.2 修改题目表
```sql
-- 添加分类字段
ALTER TABLE problem 
ADD COLUMN category_id BIGINT COMMENT '题目分类ID',
ADD COLUMN sub_category_id BIGINT COMMENT '二级分类ID',
ADD INDEX idx_category_id (category_id),
ADD INDEX idx_sub_category_id (sub_category_id);

-- 添加外键约束
ALTER TABLE problem
ADD CONSTRAINT fk_problem_category 
FOREIGN KEY (category_id) REFERENCES problem_category(id) ON DELETE SET NULL;

ALTER TABLE problem
ADD CONSTRAINT fk_problem_sub_category 
FOREIGN KEY (sub_category_id) REFERENCES problem_category(id) ON DELETE SET NULL;
```

## 七、优势分析

### 7.1 对用户
- **系统学习**：可以按知识领域系统学习，不再碎片化
- **跨语言对比**：可以对比不同语言的实现方式，加深理解
- **精准查找**：可以按分类快速找到目标题目

### 7.2 对系统
- **数据精简**：消除了重复题目，减少数据冗余
- **易于维护**：每个题目只有一个版本，修改更方便
- **扩展性强**：新增语言只需更新`supported_languages`字段

### 7.3 对教学
- **知识体系化**：分类体系符合计算机科学知识体系
- **教学连贯性**：可以按知识领域设计教学计划
- **评估精准**：可以按领域评估学生的学习情况
