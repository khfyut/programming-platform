# 个性化在线编程与答疑系统 - 技术汇报

## 1. 项目背景与目标

### 1.1 项目背景
随着编程教育的普及，在线编程学习平台成为重要的学习工具。传统的编程学习平台存在以下问题：
- 代码运行环境配置复杂
- 题目判题反馈不够详细
- 缺乏个性化学习指导
- 编程问题难以得到及时解答

### 1.2 项目目标
本项目旨在构建一个完整的在线编程学习与AI答疑平台，具体目标包括：
- 提供安全、隔离的代码运行环境
- 实现自动化判题与详细的错误反馈
- 集成AI智能答疑功能，提供即时编程指导
- 建立个性化学习推荐系统
- 提供便捷的题库管理功能

## 3. 核心功能模块的设计思路与实现方案

### 3.1 代码运行与判题系统
**设计思路**：
- 采用Docker容器隔离技术，确保代码运行安全
- 实现多语言支持（Java、Python）
- 提供详细的编译错误和运行时错误反馈
- 支持批量测试用例执行

**实现方案**：
- `DockerUtil`类：封装Docker Java API，实现容器创建、代码执行、结果获取
- `CodeSandboxServiceImpl`类：提供代码运行服务，支持单测和批量测试
- 多线程执行：使用线程池并行处理多个测试用例，提高判题效率

**关键代码**：
```java
// DockerUtil.runCode方法核心实现

            "echo \"%s\" > Main.java && javac Main.java && java Main",

                    .withMemory(memory)

            )
            .withCmd("/bin/sh", "-c", runCommand)
            .exec().getId();
    
    // 获取执行结果
    // ...
}
```

## 4. 关键技术难点及解决方案

### 4.1 代码运行环境隔离
**技术难点**：
- 确保用户代码不会影响系统安全
- 限制代码运行资源使用
- 处理代码运行超时情况

**解决方案**：
- 使用Docker容器隔离运行环境
- 设置容器内存限制（128M）和网络隔离（none模式）
- 实现超时机制，避免死循环导致系统资源耗尽

### 4.2 高效的批量判题
**技术难点**：
- 处理多个测试用例的并行执行
- 保证判题结果的准确性和一致性
- 优化判题速度，提升用户体验

**解决方案**：
- 使用线程池并行处理测试用例
- 统一的错误处理和结果解析机制
- 优化Docker容器启动和销毁流程，减少开销

### 4.3 AI API集成与会话管理
**技术难点**：
- 处理AI API调用的网络延迟
- 管理多轮对话的上下文
- 处理API调用失败的情况

**解决方案**：
- 实现异步调用和超时处理
- 设计会话管理机制，保存对话历史
- 完善的错误处理和重试机制

### 4.4 大规模题库管理
**技术难点**：
- 高效处理批量题目导入
- 实现题目验重机制
- 优化题目查询性能

**解决方案**：
- 使用多线程处理文件解析和导入
- 基于标题和内容的相似度分析实现验重
- 为题目表添加适当的索引，优化查询性能

## 5. 开发过程中遇到的问题与应对策略

### 5.1 Docker容器管理问题
**问题**：Docker容器创建和销毁过程中出现资源泄露
**应对策略**：
- 实现容器的强制删除机制，确保异常情况下容器也能被清理
- 优化容器启动参数，减少资源消耗
- 添加容器操作的异常处理和日志记录

### 5.2 AI API调用不稳定
**问题**：AI API调用偶尔失败或响应超时
**应对策略**：
- 实现API调用的重试机制
- 添加详细的错误日志和监控
- 优化API请求参数，减少请求失败率

### 5.3 批量导入性能问题
**问题**：大批量题目导入时系统响应缓慢
**应对策略**：
- 实现异步导入机制，避免阻塞主线程
- 使用批量插入操作，减少数据库交互次数
- 优化文件解析算法，提高处理速度

### 5.4 前端与后端数据交互问题
**问题**：前后端数据格式不一致，导致交互失败
**应对策略**：
- 制定统一的API数据格式规范
- 实现前端请求拦截器和响应拦截器
- 添加详细的API文档，明确数据格式要求
