# 个性化在线编程与答疑系统论文详细提纲

## 一、文档定位

- 暂定论文题目：**个性化在线编程与答疑系统的设计与实现**
- 论文类型：本科计算机类毕业论文
- 写作主线：围绕“系统设计与实现”展开，不按算法研究论文写法组织正文
- 内容边界：正文只写当前仓库中已经实现或能够被代码、接口、页面、数据库文档证实的内容；未完全落地功能只写入“总结与展望”
- 当前交付目标：形成一份可直接扩写为论文初稿的详细提纲

## 二、写作总原则

### 2.1 术语统一

- 系统名称统一为“个性化在线编程与答疑系统”
- 架构统一表述为“Vue 3 + Vite 前端、Spring Boot 后端、MySQL 数据存储、Docker 隔离运行、JWT 认证”
- 平台定位统一表述为“面向编程学习场景的综合教学辅助平台”
- 创新点统一表述为“多模块融合、在线判题安全隔离、AI 辅导接入、学习支持能力整合”

### 2.2 写作口径

- 能在代码、接口、页面、数据库中找到对应证据的内容，写为“已实现”
- 仅在规划文档中存在、尚未形成完整实现闭环的内容，不写入“系统实现”
- 对测试指标采用“功能验证结果”或“阶段性测试结果”表述，避免写成严格性能定量结论

### 2.3 当前可直接复用的材料

- 项目总述：`README.md`、`System/README.md`
- 技术路线与模块说明：`技术汇报文档.md`
- 数据库材料：`数据库报告.md`、`数据库结构文档.md`
- 接口材料：`api-doc.md`
- 测试材料：`前端测试报告.md`、根目录各类 `test_*.py`
- 功能扩展背景：`个性化在线编程与答疑系统 新增功能产品规划.md`
- 学习路径现状说明：`学习路径重构_第一阶段完成报告.md`

## 三、论文总目录建议

1. 摘要
2. Abstract
3. 绪论
4. 相关技术与开发环境
5. 系统需求分析
6. 系统总体设计
7. 系统详细设计与实现
8. 系统测试
9. 总结与展望
