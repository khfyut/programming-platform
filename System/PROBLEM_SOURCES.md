# 题目来源指南

## 一、免费可靠的在线题库（推荐）

### 1. LeetCode 力扣
- **网站**：https://leetcode.cn/
- **特点**：
  - 题目质量高，覆盖面广
  - 有详细的题解和讨论
  - 难度分级清晰（简单/中等/困难）
  - 支持多种编程语言
- **获取方式**：
  - 手动整理题目描述和测试用例
  - 参考官方题解获取标准输出
- **适用场景**：算法题、数据结构题

### 2. 洛谷 (Luogu)
- **网站**：https://www.luogu.com.cn/
- **特点**：
  - 国内最大的OI/ACM社区
  - 题目数量多，种类丰富
  - 有完整的测试数据
  - 难度分级（入门/普及-/普及/提高-/提高/省选-/省选/NOI-）
- **获取方式**：
  - 公开题目可以直接查看
  - 部分题目有测试数据下载
- **适用场景**：竞赛题、算法题

### 3. 牛客网
- **网站**：https://www.nowcoder.com/
- **特点**：
  - 有大量企业真题
  - 题目质量高
  - 支持在线评测
- **获取方式**：
  - 题库板块公开题目
  - 剑指Offer系列
- **适用场景**：面试题、算法题

### 4. Codeforces
- **网站**：https://codeforces.com/
- **特点**：
  - 国际知名的竞赛平台
  - 题目难度高，质量好
  - 有完整的测试数据
- **获取方式**：
  - 比赛题目公开
  - 可下载测试数据
- **适用场景**：高级算法题、竞赛题

## 四、自建题目注意事项

### 1. 题目设计原则
- **清晰性**：题目描述要清晰，避免歧义
- **完整性**：提供输入输出格式说明
- **测试用例**：
  - 至少1个样例用例（展示用）
  - 3-5个正式用例（判题用）
  - 包含边界条件测试
  - 包含大数据测试

### 2. 版权问题
⚠️ **重要提醒**：
- 直接复制其他平台的完整题目可能涉及版权问题
- 建议：
  - 参考题目思路，自己重新描述
  - 使用公开的、无版权限制的题目
  - 注明题目来源
  - 仅用于学习和毕业设计，不做商业用途

### 3. 题目质量检查清单
- [ ] 题目描述清晰易懂
- [ ] 有明确的输入输出格式
- [ ] 有样例输入输出
- [ ] 有多个测试用例
- [ ] 测试用例覆盖各种情况（正常、边界、异常）
- [ ] 难度分级合理
- [ ] 有对应的知识点标签

## 五、批量导入建议

### 1. 数据准备流程
1. 选择合适的题目来源
2. 整理题目信息（标题、描述、输入输出、难度等）
3. 设计测试用例
4. 使用Excel/CSV模板整理
5. 通过导入接口批量导入

### 2. Excel模板使用
参考 `IMPORT_TEMPLATE.md` 中的模板格式：

| 标题 | 内容 | 输入示例 | 输出示例 | 难度 | 语言 | 时间限制(ms) | 内存限制(MB) | 标签 | 知识点 | 提示 | 示例解释 |
|:----:|:----:|:----:|:----:|:----:|:----:|:----:|:----:|:----:|:----:|:----:|:----:|

### 3. 测试用例单独管理
- 题目导入后，通过API单独添加测试用例
- 或直接在数据库中插入测试用例（参考 `init_sample_data.sql`）

## 六、推荐题目分类

### 入门级（10-20道）
- 输入输出练习
- 基本运算
- 条件判断
- 简单循环

### 中级（20-30道）
- 数组操作
- 字符串处理
- 排序算法
- 简单数据结构

### 高级（10-15道）
- 动态规划
- 图算法
- 搜索算法
- 高级数据结构

## 七、获取测试用例的技巧

1. **手动设计**：
   - 根据题目要求设计典型输入
   - 考虑边界情况（空输入、最大值、最小值等）
   - 设计特殊情况测试

2. **参考已有代码**：
   - 先写出正确的标程
   - 用标程生成预期输出

3. **在线验证**：
   - 在其他OJ平台提交测试
   - 验证你的测试用例是否正确
